# UX Foundations From IDF

Use this reference for beginner-friendly UX framing, product discovery, usability reviews, user interviews, research planning, mobile web UX, interaction design, and information visualization.

Source basis: `Basics-of-ux-design Interaction Design Foundation.pdf`, Interaction Design Foundation, 73-page ebook extracted locally on 2026-06-17. This file is a compact paraphrase for skill use, not a copy of the ebook.

## UX Is Broader Than Usability

UX is about how people feel when they use a product or service. Usability is a crucial part of that, but UX also includes trust, value, emotional response, accessibility, findability, and fit with user goals.

When explaining UX to nontechnical collaborators, start with three plain questions:

- Does this give the user real value?
- Can the user understand and use it without unnecessary effort?
- Does the user feel confident or satisfied while using it?

## Seven UX Factors

Use these as a broad diagnostic before applying narrower laws:

| Factor | Review Question |
|---|---|
| Useful | Does it solve a real user need or provide meaningful value? |
| Usable | Can specified users complete specified goals effectively, efficiently, and satisfactorily? |
| Findable | Can users locate the product, content, feature, or next step quickly? |
| Credible | Does the product feel trustworthy, accurate, and fit for purpose? |
| Desirable | Do branding, aesthetics, tone, and emotional design make people want to use or share it? |
| Accessible | Can people with different abilities, devices, and contexts access the experience? |
| Valuable | Does it create value for both users and the organization? |

If a review feels stuck in surface polish, run this factor check to reveal whether the real problem is value, trust, findability, or accessibility.

## Usability 5E Check

Adapted from Whitney Quesenberry's usability framing:

- Effectiveness: users can complete the right task accurately.
- Efficiency: users can complete the task without excessive steps, waiting, or input.
- Engagement: the experience feels appropriate, readable, and pleasant enough for the task.
- Error tolerance: the interface prevents likely mistakes and supports undo, redo, retry, and recovery.
- Ease of learning: first-time and returning users can build understanding without memorizing hidden rules.

Use the 5E check for forms, onboarding, dashboards, checkout, settings, admin workflows, and any flow where "easy to use" is too vague.

## Design Thinking Reminder

Design thinking is iterative and human-centered. Treat its modes as flexible, not as a rigid waterfall:

- Empathize: understand users, context, needs, constraints, and pain.
- Define: turn research into a clear problem statement and success criteria.
- Ideate: generate multiple solution directions before narrowing.
- Prototype: make ideas testable at the cheapest useful fidelity.
- Test: learn from users, evidence, and behavior; then loop back.

When a request jumps straight to UI polish, check whether the problem has actually been defined. If not, state the assumption and make the next evidence step explicit.

## User Interviews

Use interviews to learn goals, behavior, vocabulary, context, and pain points. Good interviews are prepared but conversational.

Checklist:

- Define the learning goal before writing questions.
- Prefer open questions about past behavior over leading questions about imagined preferences.
- Use a script for consistency, but allow useful follow-ups.
- Keep the participant comfortable and respect the promised time.
- Record when appropriate so the interviewer can stay present.
- Thank participants and leave space for their questions.

Avoid turning interviews into feature validation theater. If the participant is only approving your idea, the research is probably too leading.

## Research Technique Fit

Pick research methods by the decision you need to make:

| Need | Useful Methods |
|---|---|
| Understand user context | Interviews, contextual inquiry, diary studies |
| Find usability breakdowns | Usability testing, remote usability testing, first-click testing |
| Compare options | Preference testing, A/B testing, prototype tests |
| Structure content | Card sorting, tree testing |
| Represent target users | Personas synthesized from real research |
| Track behavior at scale | Analytics, funnels, event logs, support data |

No single method is universal. Each method should answer a distinct question.

## Interaction Design Five Dimensions

Review interaction design across five dimensions:

- Words: labels, instructions, errors, microcopy, button text.
- Visual representations: layout, icons, typography, color, hierarchy.
- Physical objects or space: device, pointer, keyboard, touch, environment.
- Time: motion, feedback, loading, transitions, progress, state changes.
- Behavior: what the system does, how users act, and how both respond to each other.

This is especially useful when a critique over-focuses on static screens. A usable interface is not only what it looks like; it is also what happens over time.

## Mobile Web Fundamentals

Mobile design is constrained by small screens, touch input, interruptions, unstable connections, and changing context.

Check:

- Design mobile-first when mobile use is important.
- Keep navigation shallow, clear, and prioritized by real use.
- Reduce input effort; use sensible defaults and alternate input where useful.
- Keep content concise without making the mobile experience feel second-class.
- Preserve user data when connection breaks.
- Minimize page weight, unnecessary images, and bandwidth-heavy third-party scripts.
- Maintain continuity between mobile and desktop: same account state, brand trust, and core tasks.

## Information Visualization

Information visualization should make data useful for a user's purpose. It can support:

- Presentation: explain or persuade.
- Exploration: reveal possible patterns or relationships.
- Confirmation: test whether a suspected relationship appears in data.

Review questions:

- What decision or understanding should this visualization support?
- Is the chosen representation better than plain text or a table for this task?
- What data is included, omitted, emphasized, or distorted?
- Are labels, scales, legends, colors, and comparisons honest and readable?
- Is there a table or text alternative when accessibility or precision matters?

Treat persuasive charts ethically. A visualization can clarify, but it can also mislead by selective framing.
