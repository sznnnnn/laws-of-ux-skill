# UX Research Sources

Use these sources for the daily UX research intake. Prioritize user-specified sources, then supplement with durable public sources and current product examples.

## User-Specified Learning Sources

- Type: course notes / structured UX curriculum
- Priority: high
- Access note: this may be a private Notion page. If the page cannot be read directly, do not invent its contents. Use the title as the learning theme and supplement with public Google/Coursera course descriptions, official Google design guidance, and durable UX references.
- Intake focus:
  - UX process: empathize, define, ideate, prototype, test.
  - Research methods: interviews, surveys, usability studies, personas, journey maps.

  - Portfolio/case-study structure: problem, process, evidence, iteration, outcome.

### Apple Design

- Source URL: https://www.notion.so/Apple-Design-20f886bc60ff81f3af10d35ea8fd2a38?source=copy_link
- Type: design notes / platform design philosophy
- Priority: high
- Access note: this may be a private Notion page. If the page cannot be read directly, do not invent its contents. Use the title as the learning theme and supplement with Apple Human Interface Guidelines, Apple Design resources, WWDC design sessions, and durable public critiques/case studies.
- Intake focus:
  - Clarity, deference, depth, feedback, consistency, and platform convention.
  - Human Interface Guidelines patterns for navigation, controls, layout, typography, color, motion, and accessibility.
  - Product taste: restraint, progressive disclosure, direct manipulation, and careful defaults.
  - Anti-patterns: ornamental polish that hides weak information architecture or unclear actions.

## Durable Public Sources

- Nielsen Norman Group articles and reports.
- Baymard Institute public UX research and ecommerce examples.
- W3C/WCAG accessibility guidance.
- Apple Human Interface Guidelines.
- Google Material Design guidance.
- Product changelogs and design system documentation from credible products.
- Public postmortems, teardown articles, and well-documented case studies.

## Source Handling Rules

- Prefer primary sources and dated public references.
- Keep notes compact; save the link, lesson, and application.
- Do not copy full articles or course materials.
- Label private or inaccessible sources clearly instead of guessing.
- If the user wants the agent to learn from private Notion pages, use an already-authenticated local browser/session, a user-provided export, or a proper Notion API integration. Do not request, store, or handle raw passwords.
