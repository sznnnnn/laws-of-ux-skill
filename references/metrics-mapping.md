# UX Laws -> Product Metrics Mapping

Use this table to connect UX recommendations with measurable outcomes.

## Core Mapping

- Hick's Law
  - Primary metrics: time-to-decision, drop-off at choice steps
  - Secondary metrics: filter usage, option expansion rate
- Fitts's Law
  - Primary metrics: mis-tap rate, action completion time
  - Secondary metrics: rage-click rate, backtrack after tap
- Jakob's Law
  - Primary metrics: first-time task success rate
  - Secondary metrics: help-center visits, onboarding replays
- Doherty Threshold
  - Primary metrics: interaction latency, repeated click rate
  - Secondary metrics: abandonment during loading
- Tesler's Law
  - Primary metrics: task completion rate for advanced tasks
  - Secondary metrics: usage split between default and advanced modes
- Zeigarnik / Goal-Gradient
  - Primary metrics: step completion rate, return-to-task rate
  - Secondary metrics: draft recovery rate
- Von Restorff
  - Primary metrics: primary CTA click-through
  - Secondary metrics: distraction clicks on non-primary elements
- Occam / Pareto
  - Primary metrics: usage concentration on top flows
  - Secondary metrics: maintenance complexity proxy (support tickets per feature)

## Measurement Notes

- Prefer pre/post comparison on the same segment.
- Pair one behavioral metric with one quality guardrail.
- For major IA changes, validate with both task success and time-on-task.
