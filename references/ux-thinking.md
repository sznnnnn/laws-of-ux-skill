# UX Thinking Frame

Use this reference when the task needs the user's broader Laws of UX philosophy, not just individual laws.

## Definition

Laws of UX is a collection of psychological and behavioral design principles that helps designers understand users' perception, cognition, and behavioral responses when interacting with interfaces.

It is both:

- A reference framework for design decisions.
- A systematic tool for user experience thinking.

## What It Covers

- Classic psychological laws: Hick's Law, Fitts's Law, Jakob's Law, and other principles that explain decision speed, target acquisition, and familiar interaction patterns.
- Attention and memory mechanisms: Zeigarnik Effect, Von Restorff Effect, Working Memory, Selective Attention, Serial Position Effect.
- Cognitive load and behavioral motivation: Peak-End Rule, Cognitive Load, Goal-Gradient Effect, Flow, Choice Overload.
- Visual and interaction guidance: consistency, grouping, progressive disclosure, common region, proximity, similarity, connectedness, and simplicity.

## Design Purpose

Apply these laws to:

- Understand the psychological mechanisms behind user behavior.
- Make more scientific and explainable design decisions.
- Optimize interface structure and interaction details.
- Reduce operational burden and cognitive effort.
- Improve usability, appeal, satisfaction, and retention.
- Convert design experience into reusable knowledge.

## Practical Stance

Do not use UX laws as decorative citations. Use them as decision tools:

- Diagnose the user's likely perception, attention, memory, and action path.
- Identify the strongest source of friction or uncertainty.
- Translate the principle into layout, copy, control behavior, defaults, feedback, constraints, and recovery paths.
- Keep recommendations concrete enough to implement and verify.
